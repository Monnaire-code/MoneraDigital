# Issue tracker: GitHub

Issues and implementation tickets live in this repository's GitHub Issues. Use the `gh` CLI for issue creation, reading, comments, labels, and closing.

## Conventions

- Infer `Monnaire-code/MoneraDigital` from the current Git remote.
- Create one GitHub issue per implementation ticket.
- Publish blockers before the tickets they block.
- Prefer GitHub native issue dependencies; when unavailable, record dependencies in a `Blocked by` section.
- Apply the `ready-for-agent` label only after a ticket is fully specified and unblocked.

## Pull requests as a triage surface

**PRs as a request surface: no.**
